# 2.3 前饋神經網路 (Feed-Forward Neural Network, FFN)

在 Transformer 結構中，每個 Encoder 和 Decoder 層都包含一個前饋神經網路 (Feed-Forward Network, FFN)。雖然 Attention 機制負責捕捉 token 之間的全局關係，但 FFN 則負責對每個位置的表示進行非線性轉換與特徵提取。

## 1. FFN 的基本結構

Transformer 中的 FFN 實際上是一個**位置別的 (Position-wise)** 全連接層。這意味著對於序列中的每個 token $x_i$，FFN 都會獨立地應用相同的操作。

### 1.1 經典結構 (ReLU FFN)
一個典型的經典 FFN 由兩個線性變換和一個非線性激活函數組成：

$$
\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2
$$

### 1.2 現代變體 (SwiGLU FFN)
現代模型（如 Llama 系列）通常使用 SwiGLU 結構，其將擴展層拆分為並行的兩個分支，並加入門控機制 (Gating Mechanism) 與 RMSNorm：

$$
\text{FFN}_{\text{SwiGLU}}(x) = (\text{Swish}(xW_{gate}) \otimes (xW_{up}))W_{down}
$$

其中：
- **$W_{gate}$ (Gate)**：與輸入進行投影，決定特徵通過的門控值。
- **$W_{up}$ (Up)**：將特徵擴展到高維空間的投影矩陣。
- **$\otimes$**：元素級乘法 (Element-wise product)。
- **$\text{Swish}$**：常用的激活函數。
- **$W_{down}$ (Down)**：將特徵壓縮回原始維度的投影矩陣。
- **RMSNorm**：在輸入這些層之前，通常會使用 RMSNorm (對應 `ffn_norm.weight`) 來穩定激活值。

## 2. 為什麼需要 FFN？

如果 Transformer 只有 Attention 層，模型將僅僅是在對輸入向量進行加權平均（線性組合）。FFN 的引入提供了以下關鍵能力：

1. **非線性能力 (Non-linearity)**：激活函數允許模型學習複雜的非線性映射，這是處理自然語言複雜語義的基礎。
2. **特徵擴展與壓縮 (Expansion & Compression)**：
   - **擴展層 ($W_1$)**：將特徵投影到更高維的空間，讓模型能更精細地分離特徵。
   - **壓縮層 ($W_2$)**：將提取後的特徵重新整合回原始維度，以便傳遞給下一層。
3. **知識儲存 (Knowledge Storage)**：研究表明，FFN 層在 LLMs 中扮演著類似「鍵值記憶體 (Key-Value Memory)」的角色，儲存了大量事實性知識。

## 4. 與 TurboQuant 的關聯

在 TurboQuant 的量化過程中，FFN 的權重矩陣 $W_1$ 和 $W_2$ 佔據了模型參數的絕大部分。對這些矩陣進行高效量化（例如利用 PolarQuant 或 QJL）能顯著降低模型權重對記憶體的佔用，同時在推理時保持精準度。
